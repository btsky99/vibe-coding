# 🧐 코드 리뷰 프로토콜 (Code Review)

이 스킬은 코드를 **4가지 관점**에서 체계적으로 검토합니다. 칭찬보다 **개선이 필요한 지점**을 찾는 것이 목표입니다.

## 리뷰 체크리스트

### 🔒 1. 보안 (OWASP Top 10 기준)

- [ ] SQL Injection: 사용자 입력이 직접 쿼리에 포함되지 않는가?
- [ ] XSS: 사용자 입력이 HTML에 그대로 출력되지 않는가?
- [ ] 인증/인가: API 엔드포인트에 적절한 인증이 있는가?
- [ ] 민감 정보: 비밀번호, API 키가 로그나 응답에 노출되지 않는가?
- [ ] 경로 탐색: 파일 경로 관련 입력이 검증되는가?

### ⚡ 2. 성능

- [ ] N+1 쿼리: 루프 내 DB 쿼리가 없는가?
- [ ] 불필요한 재렌더링: React 컴포넌트가 과도하게 리렌더되지 않는가?
- [ ] 메모리 누수: 이벤트 리스너, 타이머 정리가 되는가?
- [ ] 캐싱: 반복 계산 결과를 캐싱하고 있는가?

### 🏗️ 3. 코드 품질

- [ ] 단일 책임: 함수/클래스가 하나의 역할만 하는가?
- [ ] 중복 코드: DRY 원칙을 지키는가?
- [ ] 매직 넘버: 숫자 상수가 이름 있는 변수로 대체되었는가?
- [ ] 에러 처리: 예외 상황이 적절히 처리되는가?

### 📖 4. 가독성

- [ ] 이름: 변수/함수명이 역할을 명확히 나타내는가?
- [ ] 주석: 복잡한 로직에 '왜(Why)' 주석이 있는가?
- [ ] 함수 길이: 함수가 50줄 이하인가?

## 리뷰 결과 형식

```
🔴 Critical (즉시 수정 필요): ...
🟡 Warning (수정 권장): ...
🔵 Info (참고 사항): ...
✅ Good (잘 된 점): ...
```

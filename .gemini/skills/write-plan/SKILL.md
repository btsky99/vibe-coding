# 📝 구현 계획 작성 프로토콜 (Write Plan)

이 스킬은 승인된 아이디어를 **실행 가능한 마이크로태스크**로 분해합니다. 계획 없이 코드를 작성하지 마십시오.

## 1단계: 컨텍스트 파악

1. **현재 코드베이스 스캔**: 수정할 파일들을 읽고 기존 패턴과 구조를 파악합니다.
2. **의존성 확인**: 필요한 패키지가 이미 설치되어 있는지 확인합니다.
3. **충돌 지점 파악**: `.ai_monitor/data/task_logs.jsonl`을 읽어 다른 에이전트가 같은 파일을 수정 중인지 확인합니다.

## 2단계: 계획 분해 (Task Breakdown)

각 태스크는 다음 형식을 따릅니다:

```
[ ] Task N: {동사} + {대상} + {목적}
    - 파일: {수정/생성할 파일 경로}
    - 방법: {구체적인 구현 방법 1~2줄}
    - 검증: {완료 여부를 확인하는 방법}
    - 예상 위험: {발생 가능한 문제}
```

**태스크 분해 원칙:**
- 하나의 태스크는 **30분 내 완료 가능**한 크기로 분해합니다.
- 태스크 간 의존성을 명시합니다. (예: "Task 3은 Task 1 완료 후 시작")
- 백엔드 변경 → 프론트엔드 변경 → 테스트 순서를 지킵니다.

## 3단계: 계획 저장

1. `ai_monitor_plan.md` 파일에 계획을 저장합니다.
2. 사용자에게 계획 전문을 보여주고 승인을 요청합니다.
3. **승인 전 코드 작성 금지.**

## 4단계: 실행 핸드오프

승인 후 `execute-plan` 스킬을 호출하여 계획을 실행합니다.

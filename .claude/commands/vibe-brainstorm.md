# 🧠 브레인스토밍 워크플로우 (Intelligence Memory Integration)

**절대 원칙: 사용자 승인 전 코드를 한 줄도 작성하지 않습니다.**

## 7단계 절차

1. **메모리 및 컨텍스트 분석**: `memory.md`를 읽어 과거 유사 사례나 결정 사항 확인
2. **명확화 질문**: 의도를 묻는 질문을 한 번에 하나씩, 선택지(A/B/C) 제공
3. **접근법 제안**: 2~3가지 방식과 각각의 장단점 제시
4. **상세 및 메모리 설계**: 선택된 방식의 파일 단위 설계와 함께, **작업 완료 후 기록할 메모리 내용**도 미리 정의
5. **승인 요청**: "이 설계대로 진행할까요?" — 명시적 승인 대기
6. **지식 기록**: 승인된 설계의 핵심 개념을 `memory.md`에 "결정된 아키텍처"로 즉시 기록
7. **핸드오프**: `/vibe:write-plan`으로 이동

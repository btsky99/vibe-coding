# 🧪 TDD 프로토콜

**원칙: 코드 전에 테스트 먼저.**

## 사이클

### 🔴 RED — 실패하는 테스트 작성
- 기대 동작을 테스트로 먼저 작성
- 테스트가 실패하는 것 확인 (당연히 실패해야 함)

### 🟢 GREEN — 최소 코드로 통과
- 테스트를 통과시키는 가장 단순한 코드 작성
- 깔끔하지 않아도 됨, 통과가 목표

### 🔵 REFACTOR — 정리
- 테스트 GREEN 상태 유지하며 코드 정리
- 중복 제거, 명확한 이름, 단일 책임 원칙

## 테스트 이름 형식
`test_{기능}_{조건}_{기대결과}`

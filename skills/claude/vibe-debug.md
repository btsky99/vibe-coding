# 🔍 지능형 디버깅 프로토콜

**원칙: 즉시 수정하려 하지 말고 근본 원인을 먼저 찾습니다.**

## 4단계 분석

### 1️⃣ 증상 수집
- 에러 메시지 전문, 스택 트레이스 확인
- `git log --oneline -10`으로 최근 변경 이력 확인
- 재현 조건 파악 (항상 발생? 간헐적?)

### 2️⃣ 가설 수립
- 원인 후보 3가지 이상 나열
- 가능성 높은 순서로 정렬
- 사용자에게 "A부터 검증하겠습니다" 보고

### 3️⃣ 검증
- 최소 재현 코드 작성
- 가설 A → 확인/반박 → 가설 B 순서
- 전략적 위치에 로그 추가

### 4️⃣ 수정 및 확인
- **최소 변경 원칙**: 근본 원인만 수정
- 수정 후 동일 조건으로 재현 테스트
- 커밋 메시지에 근본 원인 반드시 기록
